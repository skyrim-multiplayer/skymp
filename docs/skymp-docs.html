SKYMP Docs

Complete developer reference for Skyrim Multiplayer — server gamemodes, client scripting, Skyrim Platform plugins, and more.

TypeScript / JavaScript C++ Native Node.js Server Skyrim Platform Client RakNet UDP

Introduction

Skyrim Multiplayer (SKYMP) is an open-source multiplayer framework for Skyrim Special Edition. It runs a headless Node.js server (with native C++ extensions) and a Skyrim Platform-based client mod. Gamemodes are written in TypeScript/JavaScript.

Server

Node.js + C++ native module (scam_native). Hosts the world state, actor synchronization, inventories, and all game logic. Communicates via RakNet UDP.

Client

A Skyrim Platform plugin that runs inside Skyrim SE. Synchronizes movement, animations, properties, and sends/receives packets from the server.

Skyrim Platform

A modding framework for Skyrim SE that exposes the full Papyrus API to JavaScript/TypeScript. Used to build both the SKYMP client and standalone mods.

Gamemodes

Server-side scripts (single gamemode.js file or directory with index.js) that define game rules, spawn logic, events, and custom properties.

Server Setup

Requirements

  • Windows 8.1 / Windows Server 2012 or higher (64-bit)
  • Node.js 17.x
  • Visual Studio 2022 build tools (for building from source)
  • CMake 3.19+
  • Skyrim Special Edition game files (ESMs)

Building from Source

# Create build directory and configure
mkdir build && cd build
cmake .. -DSKYRIM_DIR="C:/Path/To/Skyrim Special Edition"

# Build
cmake --build . --config Release

Output is placed in build/dist/server/. Launch with launch_server.bat.

Running the Server

  1. Build the project (see above) or download a release build.
  2. Copy your Skyrim ESM files into data/ (Skyrim.esm, Update.esm, Dawnguard.esm, etc.).
  3. Edit server-settings.json with your IP address, server name, and load order.
  4. Place your gamemode.js in the server directory.
  5. Run launch_server.bat.
Your server will appear at https://skymp.io/api/servers once configured with a public IP.

Server API — mp Object

All server-side gamemode code accesses the engine through the globally available mp object. It is injected by the runtime and does not need to be imported.

SERVER mp.makeProperty(propertyName, options)

Creates a new property attached to every MpActor and MpObjectReference. Values are persisted to the database automatically. The updateOwner and updateNeighbor strings are JavaScript code executed on the client every time the value changes.

Signature

interface MakePropertyOptions {
  isVisibleByOwner: boolean;   // Player sees their own value
  isVisibleByNeighbors: boolean; // Others see this actor's value
  updateOwner: string;          // JS code run on owner's client
  updateNeighbor: string;       // JS code run on neighbor's client
}

mp.makeProperty(propertyName: string, options: MakePropertyOptions): void;

Example — Sync Player Level

mp.makeProperty("playerLevel", {
  isVisibleByOwner: true,
  isVisibleByNeighbors: false,
  updateOwner: "ctx.sp.Game.setPlayerLevel(ctx.value)",
  updateNeighbor: ""
});

Example — Sync Custom Display Name

mp.makeProperty("displayName", {
  isVisibleByOwner: true,
  isVisibleByNeighbors: true,
  updateOwner: "",
  updateNeighbor: "ctx.sp.printConsole('Neighbor name: ' + ctx.value)"
});
SERVER mp.makeEventSource(eventName, functionBody)

Creates a custom event source. The functionBody is client-side JS that calls ctx.sendEvent(data?) to notify the server. Custom event names must start with an underscore (_myEvent). The server handler is assigned via mp._myEvent = function(pcFormId) { ... }.

Signature

mp.makeEventSource(eventName: string, functionBody: string): void;

Example — Detect Local Death

mp.makeEventSource("_onLocalDeath", `
  ctx.sp.on("update", () => {
    const pl = ctx.sp.Game.getPlayer();
    const isDead = pl.getActorValuePercentage("health") === 0;
    if (ctx.state.wasDead !== isDead) {
      if (isDead) ctx.sendEvent();
      ctx.state.wasDead = isDead;
    }
  });
`);

mp._onLocalDeath = function(pcFormId) {
  console.log("Player died:", pcFormId);
  // Handle respawn, score deduction, etc.
};

Example — Send Custom Data with Event

mp.makeEventSource("_onPlayerJump", `
  ctx.sp.on("update", () => {
    const pl = ctx.sp.Game.getPlayer();
    const isJumping = pl.isInMidair();
    if (isJumping && !ctx.state.wasJumping) {
      ctx.sendEvent({ height: pl.getPositionZ() });
    }
    ctx.state.wasJumping = isJumping;
  });
`);

mp._onPlayerJump = function(pcFormId, data) {
  console.log(`Player ${pcFormId} jumped from height ${data.height}`);
};
SERVER mp.get(formId, propertyName)

Returns the current value of a property on an actor or object reference. Returns undefined if no value is set.

mp.get(formId: number, propertyName: string): any;
const pos = mp.get(0xff000000, "pos");           // [x, y, z]
const type = mp.get(0xff000000, "type");         // "MpActor" or "MpObjectReference"
const level = mp.get(0xff000000, "playerLevel"); // custom property
const online = mp.get(0xff000000, "isOnline");   // boolean
SERVER mp.set(formId, propertyName, newValue)

Sets a property value. This triggers synchronization to clients according to isVisibleByOwner / isVisibleByNeighbors flags. Only modifiable built-in properties and custom properties can be set.

mp.set(formId: number, propertyName: string, newValue: any): void;
mp.set(0xff000000, "pos", [0, 0, 0]);            // Teleport actor
mp.set(0xff000000, "isDisabled", true);         // Disable object
mp.set(0xff000000, "playerLevel", 50);          // Custom property
mp.set(0xff000000, "worldOrCellDesc", "3c:Skyrim.esm");
SERVER mp.clear()

Clears all registered properties and event sources. Typically called before reloading gamemode code.

mp.clear(): void;
mp.clear(); // Reset gamemode state

Events System

The events system allows handling game situations server-side. Built-in events fire automatically; custom events are declared with mp.makeEventSource().

// Step 1: Define the event source (client-side detection code)
mp.makeEventSource("_onSomeEvent", `
  ctx.sp.once("update", () => {
    ctx.sendEvent({ myData: 42 });
  });
`);

// Step 2: Handle it server-side
mp._onSomeEvent = (pcFormId, data) => {
  console.log("Handled for", pcFormId, "data:", data);
};
Custom event names MUST start with an underscore: _myEvent. Names without underscores are reserved for built-in events.

System Interface

The server is organized into Systems — modular TypeScript classes that implement lifecycle hooks. Each system handles a specific concern (login, metrics, spawn, etc.).

import { EventEmitter } from "events";
import { ScampServer } from "../scampNative";

interface SystemContext {
  svr: ScampServer;   // Native server instance
  gm: EventEmitter;   // Gamemode event emitter
}

interface System {
  systemName: string;

  // Called once when the server starts
  initAsync?(ctx: SystemContext): Promise<void>;

  // Called every tick
  updateAsync?(ctx: SystemContext): Promise<void>;

  // Called when a player connects
  connect?(userId: number, ctx: SystemContext): void;

  // Called when a player disconnects
  disconnect?(userId: number, ctx: SystemContext): void;

  // Called when a custom packet is received
  customPacket?(userId: number, type: string, content: Record<string, any>, ctx: SystemContext): void;
}

Example Custom System

import { System, SystemContext } from "./system";

export class MySystem implements System {
  systemName = "MySystem";

  async initAsync(ctx: SystemContext) {
    console.log("MySystem initialized");
  }

  connect(userId: number, ctx: SystemContext) {
    const actorId = ctx.svr.getUserActor(userId);
    console.log(`User ${userId} connected, actor: ${actorId}`);
  }

  disconnect(userId: number, ctx: SystemContext) {
    console.log(`User ${userId} disconnected`);
  }

  customPacket(userId: number, type: string, content: any, ctx: SystemContext) {
    if (type === "myCustomType") {
      console.log("Received custom packet:", content);
    }
  }
}

Damage & Hit Handling

When a local hit occurs, the client sends an OnHit packet to the server. The server processes it through the IDamageFormula interface.

OnHit Packet Fields

FieldTypeDescription
aggressoruint32FormId of the attacking actor

isHitBlockedboolWas the hit blocked?
isPowerAttackboolWas it a power attack?

Damage Formula

By default TES5DamageFormula is used (vanilla Skyrim formula). Implement IDamageFormula in C++ to override:

// Incoming damage calculation:
incomingDamage = isUnarmed ? raceUnarmedDamage : baseWeaponDamage;

// Armor reduction:
armorRating = sum(equippedArmorRatings) + magicArmorRating;
// fMaxArmorRating = 80, fArmorScalingFactor = 0.12 (GMST defaults)
receivedDamage = incomingDamage * 0.01 * (100 - min(armorRating * 0.12, 80));

Server-side Damage Multiplier

// In server-settings.json:
"damageMultFormulaSettings": { "multiplier": 1.5 }

Server Gamemode Setup

A gamemode is a single JavaScript file (or a directory with index.js) that uses the mp global to define game rules.

Minimal Gamemode Example

// gamemode.js

// Clear previous state (important for hot-reload)
mp.clear();

// Define a custom property synced to owner only
mp.makeProperty("coins", {
  isVisibleByOwner: true,
  isVisibleByNeighbors: false,
  updateOwner: `
    ctx.sp.printConsole("You have " + ctx.value + " coins");
  `,
  updateNeighbor: ""
});

// Detect player death and respawn them
mp.makeEventSource("_onDeath", `
  ctx.sp.on("update", () => {
    const isDead = ctx.sp.Game.getPlayer().isDead();
    if (isDead && !ctx.state.wasDead) {
      ctx.sendEvent();
    }
    ctx.state.wasDead = isDead;
  });
`);

mp._onDeath = function(pcFormId) {
  // Give 3s delay then respawn
  setTimeout(() => {
    mp.set(pcFormId, "pos", [22659, -8697, -3594]);
  }, 3000);
};

console.log("Gamemode loaded!");

TypeScript Gamemode (Advanced)

For larger gamemodes, use TypeScript with a build step:

  1. Create a directory: my-gamemode/
  2. Add package.json with "main": "dist/index.js"
  3. Write src/index.ts using mp global
  4. Compile with tsc or webpack to produce index.js
  5. Set "gamemodePath": "my-gamemode/" in server-settings.json
Set "enableGamemodeDataUpdatesBroadcast": true during local development to push gamemode changes to connected clients instantly without requiring reconnection.

Client Scripting — ctx Context

Client code runs as strings inside updateOwner and updateNeighbor of mp.makeProperty(), and inside the functionBody of mp.makeEventSource(). There are no standalone client scripts — everything is pushed from the server.

The ctx object is injected by the runtime and available in all client-side code strings.
CLIENT ctx.sp

Reference to the Skyrim Platform API. Everything available in a normal SP plugin is accessible here.

// Print to game console
ctx.sp.printConsole("Hello!");

// Get player reference
const player = ctx.sp.Game.getPlayer();

// Kill the player locally
ctx.sp.Game.getPlayer().kill();

// Set actor value
ctx.sp.Game.getPlayer().setActorValue("health", 100);
CLIENT ctx.refr

Reference to the current object being updated.

  • In makeProperty: always undefined
  • In updateOwner: same as ctx.sp.Game.getPlayer()
  • In updateNeighbor: the neighbor Actor or ObjectReference
const pos = [
  ctx.refr.getPositionX(),
  ctx.refr.getPositionY(),
  ctx.refr.getPositionZ()
];
CLIENT ctx.value

Current value of the property being processed. Is undefined in makeProperty context, or when there is no value / it's not visible.

// Set player level from server-defined value
ctx.sp.Game.setPlayerLevel(ctx.value || 1);
CLIENT ctx.state

A persistent object shared across all property update calls for this actor. Use it to store data between calls (e.g., previous state for change detection). State is shared between all properties.

// Store previous health for death detection
ctx.sp.on("update", () => {
  const hp = ctx.sp.Game.getPlayer().getActorValuePercentage("health");
  if (hp === 0 && ctx.state.prevHp !== 0) {
    ctx.sendEvent({ type: "death" });
  }
  ctx.state.prevHp = hp;
});
CLIENT ctx.get(propertyName)

Get the value of another custom property on the current actor. Built-in properties may not work correctly with this method.

const name = ctx.get("displayName");
const coins = ctx.get("coins");
CLIENT ctx.getFormIdInServerFormat() / ctx.getFormIdInClientFormat()

Convert between server-side and client-side FormIds. Server formIds use a different offset than client formIds.

// Client formId → Server formId
const serverFormId = ctx.getFormIdInServerFormat(0xff00016a);

// Server formId → Client formId
const clientFormId = ctx.getFormIdInClientFormat(0xff000000);
CLIENT ctx.sendEvent(data?)

Sends an event to the server. Only available in makeEventSource context. Optionally pass serializable data.

// Send event with no data
ctx.sendEvent();

// Send event with custom payload
ctx.sendEvent({ damage: 42, weapon: "sword" });
CLIENT ctx.respawn()

Immediately respawns ctx.refr on the client side.

ctx.respawn();

Skyrim Platform

Skyrim Platform is a modding framework that exposes the Skyrim game engine to JavaScript/TypeScript. It is used both by the SKYMP client and by standalone mod plugins.

Events — on / once / unsubscribe

import { on, once, unsubscribe } from "skyrimPlatform";

// Subscribe forever
const handle = on("equip", (event) => {
  printConsole(`Equipped: ${event.baseObj.getName()}`);
});

// Subscribe once only
once("update", () => {
  printConsole("First frame after game load");
});

// Unsubscribe later
unsubscribe(handle);

Hooks

Hooks intercept engine function calls. Currently supported: sendAnimationEvent, sendPapyrusEvent.

import { hooks, printConsole } from "skyrimPlatform";

// Hook all animation events on the player
const id = hooks.sendAnimationEvent.add({
  enter(ctx) {
    printConsole("Anim:", ctx.animEventName);
  },
  leave(ctx) {
    if (ctx.animationSucceeded) printConsole("Success!");
  }
}, /* minSelfId */ 0x14, /* maxSelfId */ 0x14, /* pattern */ "*");

// Remove hook later
hooks.sendAnimationEvent.remove(id);

// Hook Papyrus events
hooks.sendPapyrusEvent.add({
  enter(ctx) { printConsole(ctx.papyrusEventName); }
}, 0x14, 0x14, "OnSleepStart");
Always specify minSelfId, maxSelfId, and eventPattern to filter hooks at the C++ level and avoid unnecessary JS overhead.

Browser / In-Game UI

Skyrim Platform embeds a Chromium browser (CEF). Create Data/Platform/UI/index.html to load a custom UI.

import { browser, on } from "skyrimPlatform";

// Show/hide the browser overlay
browser.setVisible(true);
browser.setFocused(true); // Redirects input to browser

// Load a URL
browser.loadUrl("file:///Data/Platform/UI/index.html");
browser.loadUrl("https://example.com");
browser.loadUrl("http://localhost:1234"); // Dev server

// Run JS in the browser context
browser.executeJavaScript("document.title = 'Hello'");

// Security token
const token = browser.getToken();

// Receive messages from browser
on("browserMessage", (event) => {
  printConsole(JSON.stringify(event.arguments));
});

In HTML, send messages back to SP:

<!-- In your browser HTML -->
<button onclick="window.skyrimPlatform.sendMessage({ action: 'close' })">Close</button>

<script>
  // Works anywhere in browser context
  window.skyrimPlatform.sendMessage("hello", 42, { foo: true });
</script>

Papyrus Types

All Skyrim Papyrus types are available. Import what you need:

import { Game, Actor, Weapon, Armor, ObjectReference, Form } from "skyrimPlatform";

// Nearest actor search
const nearby = Game.findClosestActor(x, y, z, radius);

Hot Reload

Changes to files in Data/Platform/Plugins/ or Data/Platform/PluginsDev/ automatically reload all plugins without restarting Skyrim. On reload, event handlers are cleared and variables reset — except storage.

Client Plugin Setup (Skyrim Platform)

Skyrim Platform plugins are TypeScript files compiled to a single .js bundle. They run inside Skyrim SE using the ChakraCore JS engine.

File Structure

my-plugin/
  ├── package.json            ← npm config, dependencies
  ├── tsconfig.json           ← TypeScript settings
  ├── webpack.config.js       ← Bundler config
  ├── skyrim.json             ← Path to Skyrim folder
  ├── src/
  │   └── index.ts            ← Main plugin entry point
  └── build/
      └── my-plugin.js        ← Compiled output (auto-generated)

Step-by-Step Setup

  1. Install Skyrim Platform from Nexus Mods (mod ID 54909) using your mod manager.
  2. Install Node.js (LTS or Current) from nodejs.org.
  3. Copy the plugin-example folder from Data/Platform/plugin-example to somewhere outside your Skyrim directory.
  4. Open skyrim.json and set the skyrimFolder path:
    { "skyrimFolder": "C:/Program Files (x86)/Steam/steamapps/common/Skyrim Special Edition" }
  5. Open package.json and change the "name" field to your plugin name (must be unique).
  6. Edit src/index.ts with your plugin code.
  7. Run npm run dev — this installs dependencies, compiles, copies to Skyrim, and watches for changes.
  8. Launch Skyrim SE. Your plugin is now running and will hot-reload on file changes.

Plugin with Events

import { on, once, Game, Actor, Weapon, printConsole } from 'skyrimPlatform';

export let main = () => {

  // Track player health
  on('update', () => {
    const hp = Game.getPlayer().getActorValue('health');
    if (hp < 50 && !storage['warned']) {
      printConsole('Low HP!');
      storage['warned'] = true;
    }
    if (hp >= 50) storage['warned'] = false;
  });

  // React to equip events
  on('equip', (event) => {
    const weapon = Weapon.from(event.baseObj);
    if (weapon) {
      printConsole(`Equipped weapon: ${weapon.getName()}`);
    }
  });

  // Handle game load
  on('loadGame', () => {
    printConsole('Save game loaded!');
  });
};

import * as JDB from "JContainers/JDB";

const KEY = ".my-plugin.score"; // Must start with "."

  score = JDB.solveFlt(KEY, 0);

Plugin Initialization Pattern

import { on, once } from "skyrimPlatform";

export function main() {
  let allowInit = storage["my-plugin-init"] as boolean | false;

  on("loadGame", () => {
    initPlugin();           // Always init on load
    allowInit = true;
    storage["my-plugin-init"] = true;
  });

  once("update", () => {
    if (!allowInit) initPlugin(); // New game or hot-reload
  });

  function initPlugin() {
    printConsole("Plugin initialized!");
    // Set defaults, load data, etc.
  }
}

  isVisibleByOwner: true,
  isVisibleByNeighbors: false,

    if (ctx.value) {
      const form = ctx.sp.Game.getFormEx(ctx.value.formId);

Server: Detect Player Entering a Zone

mp.makeEventSource("_onZoneEnter", `
  ctx.sp.on("update", () => {
    const pl = ctx.sp.Game.getPlayer();
    const x = pl.getPositionX(), y = pl.getPositionY();
    const inZone = x > -500 && x < 500 && y > -500 && y < 500;
    if (inZone !== ctx.state.wasInZone) {
      if (inZone) ctx.sendEvent({ x, y });
      ctx.state.wasInZone = inZone;
    }
  });
`);

mp._onZoneEnter = function(pcFormId, data) {
  console.log(`Player ${pcFormId} entered zone at ${data.x}, ${data.y}`);
};
